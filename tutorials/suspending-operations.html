Suspend rendering feature

Suspend/batch operation might be useful if you want to improve the performance. Batching several costly operation can result in shorter render times, even up to two or three-fold times. By using API methods for suspending and batching you can descrese the number of reflows, it will reduce the number of render cycles for CRUD operations resulting in less layout trashing overall, less freezes, and a more responsive feel. Please note that this is advanced feature that can improve the perfromance but should be used with some caution. Misuse can result in some flickering, glitches or other visual disortion.

Suspend methods

batch

hot.batch(() => { hot.alter('insert_row', 5, 45); hot.alter('insert_col', 10, 40); hot.setDataAtCell(1, 1, 'x'); hot.setDataAtCell(2, 2, 'c'); hot.setDataAtCell(3, 3, 'v'); hot.setDataAtCell(4, 4, 'b'); hot.setDataAtCell(5, 5, 'n'); hot.selectCell(0, 0); // The table will be rendered once after executing the callback });

suspend

      
        hot.suspendRender();
        hot.alter('insert_row', 5, 45);
        hot.setDataAtCell(1, 1, 'x');
        hot.resumeRender(); // It re-renders the table internally
